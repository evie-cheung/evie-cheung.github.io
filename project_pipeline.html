📌 E-commerce Data Platform Project

🔗 Project Overview

This project demonstrates the end-to-end implementation of a lightweight data analytics pipeline using modern data tools, built to showcase practical capabilities for a real-world e-commerce dataset. The objective is to extract insights into overall sales performance, product profitability, and customer behavior.

Tech stack used: Python (Pandas), Snowflake, dbt Cloud, Power BI
Focus areas: Data cleaning, data warehousing, modeling, BI reporting

🧮 Architecture Diagram

🧬 Dataset

The source dataset contains e-commerce order records including order date, customer attributes, product details, sales, quantity, profit, discount, and shipping costs.

  • Total rows: ~10,000
  • Sample fields:
    • ORDER_DATE (Date)
    • PRODUCT, PRODUCT_CATEGORY
    • SALES, PROFIT, QUANTITY, DISCOUNT, SHIPPING_COST
    • GENDER, DEVICE_TYPE, CUSTOMER_LOGIN_TYPE

🧹 Step 1: Data Cleaning (Python)

  • ✅ Renaming columns to lowercase with underscores
  • ✅ Converting data types (e.g., dates, numbers)
  • ✅ Handling missing values (drop/replace)
  • ✅ Unifying categorical values (e.g., gender: male, female)

Output file: ecommerce_cleaned.csv

🧊 Step 2: Upload to Snowflake

The cleaned CSV was uploaded through Snowsight Web UI to the following table:

  • Database: ECOMMERCE_DB
  • Schema: PUBLIC
  • Table: ECOMMERCE_CLEANED

🏗 Step 3: Data Modeling (dbt Cloud)

Three dbt models were created:

1. fct_sales_summary

  • order_date, total_sales, total_profit, total_quantity, order_count

2. dim_product_performance

  • product, category, total_sales, total_profit, total_quantity

3. dim_customer_segmentation

  • gender, device_type, login_method, total_sales, total_orders
