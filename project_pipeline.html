E-commerce Data Platform Project

📌 Project Overview

This project demonstrates the end-to-end implementation of a lightweight data analytics pipeline using modern data tools, built to showcase practical capabilities for a real-world e-commerce dataset. The objective is to extract insights into overall sales performance, product profitability, and customer behavior.

  • Tech stack used: Python (Pandas), Snowflake, dbt Cloud, Power BI
  • Focus areas: Data cleaning, data warehousing, modeling, BI reporting

🧮 Dataset

The source dataset contains e-commerce order records including order date, customer attributes, product details, sales, quantity, profit, discount, and shipping costs.

  • Total rows: ~10,000
  • Sample fields: Order_Date, Product_Name, Category, Sales, Profit, Quantity, Discount, Customer_Segment, Gender, Device_Type, Login_Method

🧹 Step 1: Data Cleaning (Python)

  • Renamed columns to lowercase with underscores
  • Converted data types (e.g., dates, numbers)
  • Handled missing values
  • Unified categorical values (e.g., gender)

Output: ecommerce_cleaned.csv

🧊 Step 2: Upload to Snowflake

  • Database: ECOMMERCE_DB
  • Schema: PUBLIC
  • Table: ECOMMERCE_CLEANED

🏗 Step 3: Data Modeling (dbt Cloud)

Created 3 models:

  • fct_sales_summary: daily sales, profit, orders
  • dim_product_performance: product sales/profit
  • dim_customer_segmentation: orders by gender/device/login

📊 Step 4: Visualization (Power BI)

Three interactive visuals:

  • Sales Overview: KPIs + Line chart
  • Product Performance: Bar chart + Treemap
  • Customer Insights: Bar/pie charts

🧠 Key Learnings

  • Hands-on experience with Snowflake + dbt + Power BI
  • Dimensional modeling (fact/dim)
  • Full lifecycle from raw data to dashboard
